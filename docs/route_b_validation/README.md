This directory contains everything needed to reproduce the Route B controller
experiments in [`report.md`](./report.md), including the corrected 2026-07-06
OSC absolute full-pose replay result.

## How to reproduce

See `AGENTS.md` for the recommended reproduction flow. The short version:

```bash
cd docs/route_b_validation
MUJOCO_GL=egl uv run python compare_all.py    # ~5 min, writes results.json
uv run python plot_results.py                # writes figures/*.png
```

The `compare_all.py` script hardcodes the dataset path
`third_party/robomimic/datasets/can/yq/image_v15_delta_eef.hdf5` (200 demos, PickPlaceCan).
The first 5 demos are used.

To reproduce the corrected full-pose OSC absolute replay:

```bash
MUJOCO_GL=egl uv run python docs/route_b_validation/playback_eef_pose.py \
  --dataset third_party/robomimic/datasets/can/yq/image_v15_delta_eef.hdf5 \
  --all-demos \
  --osc-kp 500 \
  --osc-input-type absolute \
  --output-json outputs/route_b_validation/playback_eef_pose_all_200.json
```

Expected summary:

```text
Final success: 200/200
Mean position error: 0.51 cm
Mean orientation error: 0.39 deg
```

`playback_eef_pose.py` is the current reference for executable full-pose EEF
actions through built-in OSC. `verify_position_controller.py` is retained as a
historical position-only diagnostic and should not be used to reject full-pose
absolute OSC control.

## What the data files contain

### `results.json`

Top-level keys: plan names (`plan_A_delta_action_osc`, `plan_B1_delta_eef_osc`,
`plan_B2_absolute_osc`, `plan_C_cumulative_ik`).

Each value is a list of 5 demo results. Per demo:

```python
{
    "label": "A:demo_1",
    "n_steps": 293,
    "desired_dpos_mag_mean_cm": 1.412,   # mean |intended dpos| over the demo
    "actual_dpos_mag_mean_cm": 0.371,    # mean |actual dpos|
    "tracking_median": 0.259,             # median of actual/desired (filtered >5mm)
    "tracking_p25": ...,
    "tracking_p75": ...,
    "end_err_to_target_cm": 0.508,        # |replay_end - data.next_eef_pos[T-1]|
    "err_to_target_mean_cm": 0.57,        # mean per-step |replay - data.next_eef_pos|
    "err_to_target_max_cm": 1.23,         # max per-step
    "end_err_to_orig_cm": 0.543,          # |replay_end - data.obs_eef_pos[T-1]|
    "err_to_orig_per_step_cm": [...],     # per-step |replay - data.obs_eef_pos|
    "err_to_target_per_step_cm": [...],   # per-step |replay - data.next_eef_pos|
    "replay_traj": [[x,y,z], ...],        # (T+1, 3) EEF positions across the demo
    "data_traj": [[x,y,z], ...],          # (T, 3) EEF positions from data
}
```

### `figures/` (subdirectory)

- `err_per_step_4plans.png` — 2×2 subplots, per-plan tracking error vs step
- `err_per_step_logscale.png` — same data on log y-axis
- `err_per_step_overlay.png` — 4 plans on one axis, mean line + min-max shaded range
- `trajectory_3d.png` — 3D EEF path: data (black) vs replay (colored) for demo_1 of each plan
